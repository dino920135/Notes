- # Logseq Admonitions using "<"
	- Logseq doesn't have a built-in `<` syntax for admonitions (like Obsidian's `> [!info]`), but many users implement this using **Custom CSS**. 
	- By using a specific character like `<` at the start of a block and styling it in `logseq/custom.css`, you can create beautiful callouts.
- ## 1. Example Admonitions
	- < **TIP**
	  This is a custom tip admonition triggered by the `<` character and some CSS magic.
	- < **WARNING**
	  Be careful when editing your `custom.css` file!
	- < **INFO**
	  Logseq is very flexible with its styling.
- ## 2. How to Setup (Custom CSS)
	- To make the examples above look like colored boxes, add this to your `logseq/custom.css`:
	- ```css
	  /* Custom Admonition Style for blocks starting with < */
	  div[data-refs-self*='["<"]'] {
	    background-color: #f0f7ff;
	    border-left: 5px solid #007bff;
	    padding: 10px;
	    border-radius: 4px;
	  }
	  
	  /* Optional: Style specifically for Warning */
	  div[data-refs-self*='["<"]']:has(strong:contains("WARNING")) {
	    background-color: #fff3cd;
	    border-left-color: #ffc107;
	  }
	  ```
- ## 3. Why use "<"?
	- It is a fast, single-character trigger that doesn't conflict with standard Markdown blockquotes (`>`).
	- It stays readable in plain text editors.
	- It allows for easy filtering in queries.
- ## 4. Alternative: Native Admonitions
	- If you prefer the built-in blocks without CSS, you can use:
	- #+BEGIN_TIP
	  This is a native Logseq tip block.
	  #+END_TIP
	- #+BEGIN_IMPORTANT
	  This is a native important block.
	  #+END_IMPORTANT
